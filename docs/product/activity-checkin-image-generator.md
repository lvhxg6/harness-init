# AI 活动打卡图片生成工具 PRD

## 1. 背景与目标

公司跑步活动中存在拍照打卡任务，任务照片需要包含指定颜色和元素。用户到达活动地点后，可能因为现场条件、时间、人员或场景限制，无法快速拍到符合要求的照片。

本功能提供一个移动端 H5 生图工具。用户可以上传现场照片，也可以不上传照片直接生成北京奥林匹克森林公园场景图片。系统基于用户选择的颜色、元素和补充描述，一次生成 3 张真实风格候选图，用户可放大查看并下载。

核心目标是：快速生成符合活动要求、真实自然、适合上传打卡的图片。

## 2. 用户与使用场景

目标用户：

- 参与跑步活动、需要完成拍照打卡任务的员工或活动参与者。

典型场景：

1. 用户到达活动地点后，用手机打开 H5 页面。
2. 用户上传现场图片，或选择不上传图片。
3. 用户选择需要出现的颜色和元素。
4. 用户可填写补充描述。
5. 用户点击生成。
6. 系统生成 3 张候选图片。
7. 用户放大查看，选择满意图片下载后上传活动任务。

## 3. 范围

### 3.1 第一版包含

- 移动端 H5 页面。
- 图片上传，可选。
- 纯文生图模式。
- 图生图模式。
- 颜色多选。
- 元素多选。
- 补充描述输入。
- 图片比例选择。
- 每次生成 3 张图片。
- 结果图片放大查看。
- 单张图片下载。
- 再来一组。
- IP 级生成次数限制。

### 3.2 第一版不包含

- 登录。
- 用户中心。
- 生成历史记录。
- 支付。
- 图片收藏。
- 分享功能。
- 后台管理。
- 复杂手工 mask 编辑。

## 4. 生成模型

使用 OpenAI GPT Image 2。

模型能力要求：

- 支持纯文生图。
- 支持基于上传图片进行图像编辑或变体生成。
- 生成结果优先追求真实、自然、高质量。

推荐接口策略：

- 用户上传图片时，调用图片编辑能力。
- 用户未上传图片时，调用纯文生图能力。
- 每次固定生成 3 张图片。

## 5. 页面与交互

### 5.1 页面定位

页面是移动端优先的 H5 页面，需要适配 iPhone 和安卓主流屏幕尺寸。

视觉风格：

- 小清新。
- 简洁。
- 美观。
- 不要复杂、沉重或过度商业化。
- 主流程清晰，适合现场快速操作。

### 5.2 页面结构

页面包含：

1. 顶部标题和简短说明。
2. 图片上传区域。
3. 颜色多选区。
4. 元素多选区。
5. 图片比例选择。
6. 补充描述输入框。
7. 生成按钮。
8. 生成中状态。
9. 三张结果图片。
10. 图片放大预览。
11. 下载按钮。
12. 再来一组按钮。

### 5.3 图片上传

上传图片是可选项。

支持：

- 从相册选择。
- 手机拍照上传。

建议支持格式：

- JPG。
- PNG。
- WebP。

上传图片后应展示预览。

### 5.4 颜色选择

颜色可多选。

候选项：

- 黄色。
- 蓝色。
- 粉色。

多选规则：

- 用户选择的所有颜色都必须出现在生成图中。
- 颜色出现方式必须自然、美观。
- 颜色可以体现在衣服、帽子、鞋子、背包、运动装备、建筑装饰、路牌、花朵等合理位置。
- 不允许为了满足颜色要求而出现突兀大色块。

### 5.5 元素选择

元素可多选。

候选项：

- 人。
- 建筑。
- 风景。

多选规则：

- 用户选择的所有元素都必须出现在生成图中。
- 元素应与原图或奥森场景自然融合。

人物规则：

- 如果上传图片中已经有人，且用户选择了“人”，不要重复生成新的主体人物。
- 如果上传图片中没有人，且用户选择了“人”，可以自然加入跑者、路人或活动参与者。
- 如果图片中已有真人，应保留人物身份、姿态、位置和整体真实感。
- 不做换脸，不改变人物身份。

建筑规则：

- 建筑应符合现场环境。
- 不要生成突兀的摩天楼、城堡、科幻建筑或与场景明显冲突的建筑。

风景规则：

- 风景元素可包括树木、湖面、步道、草地、天空、自然光线等。
- 不应把原图改造成完全不同的地点。

### 5.6 补充描述

用户可以输入补充性描述。

示例：

- 傍晚自然光。
- 人站在湖边。
- 不要太多人。
- 更像手机实拍。

建议限制：

- 最多 200 字。

### 5.7 图片比例

用户可以选择图片比例。

选项：

- 跟随原图。
- 1:1。
- 4:3。
- 3:4。
- 16:9。
- 9:16。

默认：

- 有上传图时，默认跟随原图。
- 无上传图时，默认 4:3。

### 5.8 生成结果

点击生成后，系统一次生成 3 张候选图片。

每张候选图片支持：

- 缩略图展示。
- 点击放大查看。
- 下载。

结果区域提供：

- 再来一组按钮。

再来一组规则：

- 使用当前上传图片、颜色、元素、比例和补充描述重新生成 3 张图片。
- 计入 IP 生成次数限制。

## 6. 生成逻辑

### 6.1 图生图模式

触发条件：

- 用户上传了图片。

生成要求：

- 基于上传图片生成。
- 尽量保留原图主要场景、构图、光线、透视和真实感。
- 根据用户选择自然加入或强化颜色和元素。
- 结果应像真实照片，不应像插画或海报。

如果图片包含真人：

- 优先做场景增强。
- 如用户选择颜色，优先调整衣服、帽子、背包、鞋子等非身份特征物品颜色。
- 如用户选择建筑，在背景或合适位置自然加入符合场景的建筑元素。
- 如用户选择风景，增强自然景观。
- 如用户选择人，优先保留现有人物，不新增重复主体人物。

如果图片不包含真人：

- 如用户选择人，在画面合适位置自然加入跑者、路人或活动参与者。

### 6.2 纯文生图模式

触发条件：

- 用户未上传图片。

生成要求：

- 必须生成北京奥林匹克森林公园场景。
- 画面应像真实手机照片或真实活动现场照片。
- 根据用户选择包含所有颜色和所有元素。
- 不生成插画、动漫、卡通、海报风图片。

### 6.3 真实性要求

真实性是最高优先级。

生成结果不得出现：

- 明显 AI 合成感。
- 卡通、动漫、插画、海报风。
- 明显拼接痕迹。
- 水印。
- 乱码文字。
- 畸形人物。
- 多余肢体。
- 过度锐化或过度磨皮。
- 不合理透视。

## 7. IP 限制

第一版不做登录，使用 IP 做生成次数限制。

规则：

- 每个 IP 最多生成 10 次。
- 每次生成 3 张图片。
- 每个 IP 最多可获得 30 张图片。
- 点击“再来一组”也计为 1 次生成。

达到限制后：

- 不再调用生图接口。
- 页面提示用户今日生成次数已用完。

限制周期：

- 第一版按自然日限制。

## 8. 接口建议

前端 H5 不直接调用 OpenAI。

后端提供统一接口：

```http
POST /api/images/generate
Content-Type: multipart/form-data
```

请求字段：

- `image`：可选，上传图片。
- `colors`：必填，多选数组，取值为 `yellow`、`blue`、`pink`。
- `elements`：必填，多选数组，取值为 `person`、`building`、`scenery`。
- `extraPrompt`：可选，最多 200 字。
- `aspectRatio`：可选，取值为 `follow_original`、`1:1`、`4:3`、`3:4`、`16:9`、`9:16`。

响应字段：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "remainingGenerations": 9,
    "images": [
      {
        "id": "image-1",
        "url": "https://example.com/generated-1.png"
      }
    ]
  }
}
```

## 9. 错误处理

需要处理：

- 图片格式不支持。
- 图片过大。
- 颜色未选择。
- 元素未选择。
- 补充描述超长。
- 生成接口失败。
- 生成超时。
- 内容安全拦截。
- IP 生成次数超限。

错误提示应简洁明确，适合移动端展示。

## 10. 验收标准

### 10.1 功能验收

- 用户可以打开移动端 H5 页面。
- 页面在 iPhone 和安卓常见尺寸下不溢出、不遮挡。
- 用户可以上传图片并看到预览。
- 用户可以不上传图片直接生成。
- 用户可以多选颜色。
- 用户可以多选元素。
- 用户可以输入补充描述。
- 用户可以选择图片比例。
- 点击生成后，系统返回 3 张图片。
- 用户可以点击图片放大查看。
- 用户可以下载单张图片。
- 用户可以点击再来一组重新生成。
- 每个 IP 每日最多生成 10 次。
- 超限后页面给出明确提示。

### 10.2 内容验收

- 多选颜色时，所有选中颜色都应出现。
- 多选元素时，所有选中元素都应出现。
- 图生图结果应保留原图主要场景和真实感。
- 上传图已有真人时，不应重复生成新的主体人物。
- 纯文生图必须表现北京奥林匹克森林公园场景。
- 生成结果应真实自然。
- 生成结果不应明显呈现卡通、插画、动漫或海报风。
- 生成结果不应出现明显水印、乱码文字、拼接痕迹或畸形人物。

### 10.3 技术验收

- OpenAI API Key 只能在后端使用，不能暴露给前端。
- 后端根据是否上传图片选择图生图或纯文生图。
- 每次生成固定请求 3 张图片。
- IP 限制在调用生图接口前生效。
- 生成失败不应扣减次数，除非 OpenAI 已实际发起生图请求。
- 前端生成中状态清晰，防止重复提交。
